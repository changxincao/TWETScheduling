# setup 数据接入说明

这次调整只做了两件事。第一，在 `src/Common/ETConverter.java` 里，把 Tanaka 单机 TWET 数据转成并行机数据时，直接附带生成一份 sequence-dependent setup 矩阵；第二，在 `src/Basic/Data.java` 里，读取并行机实例时，识别并读入这份 setup 数据。除此之外，没有改求解逻辑，也没有改调试相关设置。

当前生成出的多机实例格式为：第一行仍然是 `n m`，接着是 `n` 行任务数据 `p d wE wT`，然后追加一段 `SETUP` 块。`SETUP` 后面共有 `n+1` 行，每行 `n+1` 个数，对应 `s_ij`。其中 `s[0][j]` 表示首任务前的 setup，`s[i][0]` 保持为 0，`s[i][i]` 也为 0。这样 `Data` 在读完任务区之后，就可以直接把 setup 矩阵填回到 `s` 里。

这次 setup 的生成方式采用固定 `eta = 0.5`，不再区分 long / short setup 两档。具体做法是先按当前实例的平均加工时间确定一个平均 setup 水平，再用截断正态随机生成任务间 setup，并用实例内容构造固定随机种子，保证同一个实例重复生成时结果一致。机器仍然按同质机处理，所以 setup 只和任务对有关，不区分机器。

本次实际覆盖的数据范围是当前已有并行机目录对应的三组算例，也就是 `data/40-2`、`data/40-4`、`data/50-2`、`data/50-4`、`data/100-2` 和 `data/100-4`。这些目录下的 `.dat` 文件已经重新生成，并带有 `SETUP` 块。
