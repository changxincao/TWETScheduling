# BPC 整体效率优化候选方向

本文记录 2026-06-21 对当前 TWET BPC 组件进行的广泛复查。目标不是再随机叠加组件，而是结合当前代码中已经实现的 ng-DSSR、completion bound、pricingOnly subtree、midpoint probe、RMIH、外包列化和 partial dominance，判断下一步哪些方向最可能带来实质效率提升。

## 1. 当前判断

当前最值得优先尝试的不是继续加强单个 labeling 小剪枝，而是两个更上层的问题：dual stabilization 和 branching 策略。前者针对列生成迭代中的 dual churn，后者针对外包列化下 root bound 已经更强、但后续几乎全靠 ArcBrancher 慢慢抬下界的问题。

文献上，branch-and-price 的慢收敛通常和主问题退化、dual 振荡、分支规则不能有效抬下界直接相关。Pessoa、Sadykov、Uchoa、Vanderbeck 关于 column generation stabilization 的工作强调，LP-based stabilization 可以显著改善列生成收敛；Baldacci、Mingozzi、Roberti 的 ng-route relaxation 和 pricing strategy 说明 ng/DSSR 更适合作为 pricing 内部的可控松弛；VRP branch-price-and-cut 综述也把 dual stabilization、heuristic pricing、ng-route、robust branching 和 cut 管理放在同一层看，而不是只靠更强 dominance。

结合当前实验，ng、completion bound、probe、pricingOnly subtree 这些内部 pricing 技术已经有用，但它们没有改变外包列化下节点多的结构原因。外包 columns 模式在多段 tariff 下 root bound 更强，但后续成功分支几乎仍全是 arc branch，每个分支带来的下界提升很小。因此继续优化单个 label 操作只能降低单节点时间，不能解决节点数问题。

## 2. 优先级最高的方向

1. dual stabilization / smoothing

这是最值得先做的全局组件。当前主问题高度退化：内部列、外包集合列、set covering 覆盖行和分段外包成本都会让 dual 在相邻 pricing round 间波动。外包列化的 root bound 不差但 pricing rounds 多，也很符合 dual churn 的表现。

实现上可以先做低侵入版本：保留真实 LP dual 作为闭合证明口径，同时维护一个 stabilized dual center。启发式 pricing、外包 pricing 和前几轮 exact pricing 可以先用
`pi_stab = alpha * pi_current + (1 - alpha) * pi_center`
定价；如果 stabilized dual 下找不到列，必须再用真实 dual 跑 exact pricing，只有真实 dual 下也无负列时才能关闭当前 RMP。这样不会破坏精确性。

验证指标应看 pricing rounds、每轮生成列数、dual 变化幅度、root 时间和节点闭合时间，而不是只看单次 exact pricing label 数。

2. 外包列化下的 branching 改进

当前 columns 模式的主要问题不是 root bound 弱，而是没有类似旧模型 tariff segment branch 的强分支。`OutsourcingMembershipBrancher` 一次只切一个 job 是否属于外包集合，太细；`ArcBrancher` 又主要作用于内部机器列，对外包集合凸组合的分数性切得很慢。

下一步更合理的是做分支评分和条件分支，而不是固定顺序。可以先记录每类候选分支的 fractional score、预测子节点下界改善、历史 pseudo-cost，然后在 columns 模式下尝试：

1. 若外包 membership 分数性明显强于 arc 分数性，则优先外包 membership；
2. 若多个 job 的外包 membership 高度相关，则尝试 group / cluster membership branch；
3. 对多段 tariff，可以重新设计更稳的 baseline / tariff-regime 分支，但不能恢复之前那种低效 baseline 区间实现。旧实现语义可行，但测试中没有减少 pricing rounds，反而更慢。

这个方向直接针对节点数，优先级高于继续微调 subtree。

3. 外包列返回策略和列池管理

当前 `OutsourcingPricingEngine` 本身很快，但返回外包集合列后会放大主问题退化和 pricing round。直接把返回上限压得太小会恶化，因为会增加重解次数；因此更可行的是“多样性受控返回”而不是简单少返回。

可以按 baseline bucket、外包 job 数、reduced cost 桶保留代表列，避免同一类外包集合一次性灌入太多。闭合时仍需要一次真实 dual 下的完整外包 pricing，保证没有漏负列。

同时，restricted master 可以做列 aging：正值列、最近基列和低 reduced-cost 列保留；长期零值且 reduced cost 高的列在当前节点 restricted set 中移除，但不从全局 pool 删除。这个对 LP 解时间和子节点继承列规模可能有直接帮助。

## 3. 中等优先级方向

1. RMIH 调用频率自适应

RMIH 是上界启发式，不影响下界证明。当前它在很多节点上会被调用，但真正改善 incumbent 的次数有限。可以改成 root 强调用、incumbent 长时间未改善时调用、gap 低于某阈值时调用，或者每隔若干节点调用。时间限制 4 秒合理，但频率还可以更保守。

2. pricing engine 调度自适应

当前顺序基本是 `HeuristicPricing -> OutsourcingPricing -> exact`。这个顺序已经比 outsourcing-first 稳，但仍可做 stagnation counter：如果某个 engine 连续多轮只生成很少列，就临时跳过；如果 exact 连续发现很多负列，则减少前置启发式调用。闭合前仍必须跑真实 dual exact pricing。

3. pseudo-cost branching

当前可先不引入复杂强分支，但可以记录每次 branch 后的实际 bound improvement，形成 brancher 和具体 arc/job 的 pseudo-cost。对 arc branch 和外包 membership branch 都有价值，尤其适合解释“root bound 强但节点多”的情况。

## 4. 暂不建议作为主线的方向

1. full SRI / LM-SRI / arc-memory SRI 默认开启

前面的实验表明，SRI 能提升 root bound，甚至让部分算例在 root 闭合，但 label 状态膨胀明显，整体求解慢。它可以保留为诊断或特定难例开关，不适合作为当前默认效率路线。

2. time-indexed graph pricing 替代当前函数 pricing

无 cut 的 time-indexed graph pricing 单次 DP 很快，但会生成大量非基本 pseudo-schedule 列，根节点列数暴涨；当前能跑动主要依赖已有 elementary/ng 启发式补充。它适合作为论文方法对照，不适合作为当前 BPC 主线替换。

3. 直接 subtree on

已有结果显示 subtree on 有时能减少部分 pricing 难度，但也可能因为列信息不足、arc fixing 过早、后续节点重构成本变高而变慢。pricingOnly subtree 比 permanent subtree 更稳，但也不是解决 columns 节点多的核心。

## 5. 建议的下一轮实验顺序

第一步做 dual stabilization 的低侵入版本。它只改变 pricing 使用的 dual，不改变主问题模型和最终闭合证明，风险最低。实验先选两个口径：普通 ng-DSSR 无外包、columns 多段 tariff。重点看 pricing rounds 是否下降。

第二步做 columns 模式 branch diagnostic 和条件分支。不要先写复杂 group branch，先把 arc、membership、machine 三类候选的分数性、历史 pseudo-cost 和 child bound improvement 统计出来，再决定是否把 membership 提前。

第三步做外包列返回多样性策略。不要再简单调小 `maxOutsourcingPricingColumns`，而是保留 baseline/cardinality/reduced-cost 多样性，同时闭合前完整 pricing。

如果这三步都没有明显改善，再考虑更重的 cut 策略或并行 pricing。

## 6. 主要参考

1. Pessoa, Sadykov, Uchoa, Vanderbeck, “Automation and Combination of Linear-Programming Based Stabilization Techniques in Column Generation”, INFORMS Journal on Computing, 2018.
2. Baldacci, Mingozzi, Roberti, “New Route Relaxation and Pricing Strategies for the Vehicle Routing Problem”, Operations Research, 2011.
3. Desaulniers 等， “Exact Branch-Price-and-Cut Algorithms for Vehicle Routing”, Transportation Science, 2019.
4. SCIP Ryan/Foster branching 文档和近年的 cluster branching 工作。它们共同说明：在列变量极多、普通变量分支弱的模型中，分支规则需要直接切到 fractional structure，而不是只依赖原始 arc。
