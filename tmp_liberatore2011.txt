Abstract The V ehicle Routing Problem with Time Windows consists of comput-
ing a minimum cost set of routes for a ﬂeet of vehicles of limited capacity visiting a
given set of customers with known demand, with the additional constraint that each
customer must be visited in a speciﬁed time window. We consider the case in which
time window constraints are relaxed into “soft” constraints, that is penalty terms are
added to the solution cost whenever a vehicle serves a customer outside of his time
window. We present a branch-and-price algorithm which is the ﬁrst exact optimization
algorithm for this problem.

1 Introduction
A common problem in distribution logistics is to decide routes and schedules of vehi-
cles that must visit a given set of customers where goods must be picked up or delivered.
The visit to each customer is often constrained to be scheduled inside a speciﬁed time
window indicated by the customer; planning algorithms for optimally routing and
scheduling the ﬂeet of vehicles must rigorously respect these restrictions. The result-
ing problem is known as the vehicle routing problem with time windows (VRPTW):
for a recent survey the reader is referred to Cordeau et al. (2001) and Kallehauge
et al. (2005). In particular branch-and-price algorithms for the VRPTW have been
developed by Desrochers et al. (1992), Desaulniers et al. (2008), Jepsen et al. (2008),
Salani (2006) and Baldacci et al. (2009).
In spite of the vast literature on the VRPTW, for practical purposes it may be
questionable whether time windows should be actually modeled by constraints like
those representing physical or logical restrictions, such as capacity constraints and
precedence constraints. In many cases time windows represent preferences about the
time at which visits should occur at customers’ locations, but violations to these pref-
erences can be allowed, provided that they are small and not too frequent. For this
purpose “soft time windows” can be deﬁned, such that if a customer is visited outside
of his preferred time window, a penalty is incurred in terms of additional costs that are
charged to the distributor/collector. In this way it is possible to compute feasible plans
that are much more profitable for the logistic operator. A more realistic model is paid
in terms of added difﬁculty in the solution process, because the optimization algorithm
must take into account the additional cost terms coming from the penalties; as shown
in the remainder of this paper this poses several computational problems that can
be tackled by suitable extensions of mathematical optimization techniques recently
developed for the VRP with additional constraints. To the best of our knowledge only
heuristic algorithms have been published so far for the VRP with soft time windows
(VRPSTW) since the early work by Sexton and Choi (1986), which concerned the
pick-up and delivery version of the problem;Koskosidis et al.(1992) proposed an opti-
mization-based heuristic which decomposes the original problem into an assignment
subproblem and a number of routing subproblems; Balakrishnan (1993) presented
several constructive heuristics; Taillard et al. (1997) and Chiang and Russell (2004)
developed tabu search heuristics; Ibaraki et al. (2005) studied acceleration techniques
for local search algorithms in the case of multiple soft time windows.
In this paper we present a branch-and-price algorithm for the exact optimiza-
tion of the VRPSTW. Since the VRPSTW turns out to be much more difﬁcult than
the VRPTW from the computational viewpoint, our exact algorithm can be used as
a heuristic providing an a posteriori approximation guarantee, when the size of the
instances at hand prevent from ﬁnding a provable optimal solution within the available
amount of computing time.
The pricing sub-problem is a resource constrained elementary shortest path problem
(RCESSP), which is NP -hard on graphs with negative cost cycles. We solve it by
bi-directional dynamic programming with decremental state space relaxation, a tech-
nique developed by Righini and Salani (2008b), with the additional difﬁculty that in
the case of soft time windows the number of non-dominated dynamic programming

3.2.1 Bounding the number of vehicles
In the VRPTW, owing to the triangle inequality, any feasible solution certainly uses
at least as many vehicles as the optimal solution. Unfortunately this is not true in
the VRPSTW, because of the penalty terms due to the soft time windows: it may be
optimal to use more vehicles than necessary, in order to pay less penalties. However
it is possible to exploit the value z
UB of a feasible solution to compute an upper limit
to the number of vehicles needed to achieve the same cost.
The cheapest route a vehicle can follow is a cycle(0, i, N + 1). Consider a list of all
these simple cycles, sorted by non-decreasing costs. Consider the maximum value h,
such that ∑h
h=1 cost (h) ≤ zUB , where cost (h) indicates the cost of the cycle in
position h in the sorted list. Each solution using more than h vehicles certainly has a
cost greater than zUB ; hence an upper bound to the number of vehicles in an optimal
solution is VUB = h.
3.3 Column management
Hereafter we describe how the RLMP is initialized and how the algorithm manages
columns dynamically generated.
3.3.1 Initialization of the RLMP
At the root node, to ensure the existence of a feasible solution to the RLMP , we insert
a dummy column, with a very high cost, representing a route that visits all customers.
This dummy column is inserted just to ensure that the initial linear program is feasible;
this is necessary to start pricing out new columns. After some iterations the dummy
column leaves the optimal basis owing to its very high cost.
For each customer i ∈ N we also insert a column representing a route which visits
only that customer.
At each node of the search tree the RLMP is initialized with the same set of col-
umns of the last considered node, with the only possible exception of the columns that
are not feasible due to previous branching decisions. Owing to the best-ﬁrst-search
strategy we adopted, the last solved node is not necessarily the predecessor of the
current node.

4 Pricing
The pricing subproblem is a resource constrained shortest path problem with soft time
windows. A vehicle must go from vertex 0 to vertex N + 1, visiting a subset of the
other vertices; no cycles are allowed. Because there are non-negative costs c associ-
ated with the arcs and non-negative prizesλassociated with the vertices, negative cost
cycles can occur. Therefore the requisite that the path must be elementary does not
come for free from cost minimization but it must be explicitly enforced. The resource
constrained elementary shortest path problem (RCESPP) on graphs with negative cost
cycles is strongly NP -hard (see Dror 1994).
The capacity constraint is taken into account as a resource constraint: each vehicle
leaving the depot has Q units of available resource and every time it visits a customer
with demand q
i it consumes qi units of resource. The objective is to minimize the cost,
given by the sum of the costs of the arcs traversed plus the sum of the penalties for
anticipation and delay, minus the sum of the prizes collected at the vertices visited.
The main difﬁculty in dealing with soft time windows in column generation algo-
rithms is that the possibility of trading time versus cost generates an inﬁnite number
of possible solutions of the pricing problem that do not dominate one another. In algo-
rithmic terms this means that the pricing algorithm, usually a dynamic programming
algorithm, must take into account an inﬁnite number of Pareto-optimal states. In this
paper we extend the ideas presented in previous papers on the RCESPP to the case with
soft time windows, implying continuous resources. In particular we present extensions
of the bi-directional search technique presented in Righini and Salani (2006), coupled
with decremental state space relaxation presented in Righini and Salani (2008a,b),

4.1 Bi-directional dynamic programming
The pricing sub-problem is solved to optimality by a bi-directional dynamic program-
ming algorithm.
4.1.1 States, labels and extension
A state associated with vertex i ∈ N represents a path from the depot 0 to i. Different
states associated with the same vertex correspond to different feasible paths reaching
that vertex.
When a vehicle reaches a vertex it can start the service immediately or it can wait
and start the service at a later time in order to reduce costs in case of early arrival.
Therefore from each feasible state an inﬁnite number of feasible states can be gener-
ated. For this reason our dynamic programming algorithm must take into account an
inﬁnite number of non-dominated states and this is done by grouping them into labels.
Each label corresponds to an inﬁnite number of states associated with the same path.
A label associated with vertex i ∈ N is a tuple L

describes the trade-off between cost and time. This function is piecewise linear and
convex, because it is the sum of piecewise linear and convex functions, like the one

4.1.5 Search policy
Labels are extended according to the vertices they are associated with. The vertices are
cyclically visited in increasing order of the starting time a
i of their soft time windows;
for each vertex the algorithm extends all its labels that have not yet been extended.
At each vertex all the labels not yet extended are sorted according to a hierarchical
criterion: ﬁrst they are sorted by increasing overall length of the path; in case of tie
they are sorted by increasing amount of capacity consumed; in case of further tie, they
are sorted by number of vertices visited.

4.1.7 Resource-based bounding
Since all forward and backward states generated by the bi-directional search algo-
rithm are tentatively joined, it is crucial to reduce their number as much as possible.
To this purpose we select a critical resource, whose consumption is monotone along
the paths, and we do not extend states in which at least half of the available amount
of that resource has already been consumed. We tried using r, T and |S| as critical
resources. These choices are compared in the next section.
4.2 Decremental state space relaxation
Decremental state space relaxation (DSSR) was independently introduced by Righini
and Salani (2008b) and by Boland et al. (2006) to reduce the number of states to be
explored by dynamic programming. The basic idea is that the elementary path con-
straints are not imposed on all the vertices of the graph but rather on a subset of critical
vertices. If the optimal solution of the pricing problem is not elementary, one or more
vertices visited more than once are inserted into the critical vertex set and the dynamic
programming algorithm is executed again. In this way at each iteration a lower bound
for the pricing problem is computed. Computational experiments reported in Righini
and Salani (2008b) show that the number of vertices that must be deﬁned as critical in
order to obtain a feasible solution is usually only a small fraction of N and therefore
the pricing problem is solved to proven optimality in a fraction of the computing time
required by dynamic programming when multiple visits are forbidden at all vertices.

In our algorithm we map each label(S, i, r, C(Ti)) onto a new label( ˆS, i, r, C(Ti)),
where ˆS is the set of critical vertices, initially empty. This is done for both forward and
backward labels. The dominance rule is then modiﬁed, replacing condition S′ ≤ S′′
by ˆS′ ≤ ˆS′′. If unreachable vertices are also considered, they are recorded in a vector
ˆU ⊆ N and the dominance rule is ˆS′ + ˆU′ ≤ ˆS′′ + ˆU′′.
4.2.1 DSSR policies
In Boland et al. (2006) several different policies to update the critical set were con-
sidered. A computational comparison between them is also reported in Righini and
Salani (2008a). In the algorithm for the VRPSTW, we used the policy called HMO
according to the terminology ofBoland et al. (2006): it consists in adding to the critical
vertex set one vertex at each iteration, selecting one at random among those visited
the largest number of times.
4.2.2 Initialization of the critical vertex set
Also the initialization of the critical vertex set can be done in different ways. InRighini
and Salani (2008a) several policies are compared. The aim is to identify a subset of
vertices that have a high probability to belong to the ﬁnal critical vertex set. Let us
deﬁne φij to be a measure of the “cycling attractiveness” of a vertex i with respect to
av e r t e xj as the ratio of the prize λi over the duration of the cycle i- j-i:
φij = λi/(θi + tij + θj + tji ).
Now we deﬁne an ordering of the vertices based on the following four criteria:
− Highest Cycling Attractiveness (HCA): order by max j∈N\{i}{φij }.
− Total Cycling Attractiveness (TCA): order by ∑
j∈N\{i} φij .
− Weighted HCA (WHCA): order by max j∈N\{i}{φij (bi − ai)}.
− Weighted TCA (WTCA): order by ∑
j∈N\{i} φij (bi − ai).
In general these criteria give different results and none of them is consistently reliable
to identify the necessary vertices to be put in the critical vertex set. Let us deﬁne
HCA m, TCA m, WHCA m and WTCA m to be the sets containing the ﬁrst m vertices
according to each of the ordering criteria above. We use as an initial critical vertex set
the one obtained from the intersection of these four sets. By a suitable choice of the
value of m the initial critical vertex set can be set with the aim of reducing the num-
ber of iterations (high m) or reducing the probability of inserting into it unnecessary
vertices (low m). In our computational experiments we set m = 8.
4.3 Heuristic pricing
It is common practice to price out columns in a heuristic way, in order to speed up the
column generation algorithm [see for instance Dell’Amico et al. (2006)]. Only when
heuristic pricing algorithms fail it is necessary to resort to exact pricing techniques.
123

64 F. Liberatore et al.
For this purpose we developed three heuristic pricing algorithms, with increasing com-
plexity and accuracy: a greedy constructive algorithm, a local search algorithm and a
heuristic dynamic programming algorithm.
Hence the overall pricing procedure consists of the following steps: ﬁrst, the col-
umn pool is searched and all columns in it are evaluated with the incumbent values
of the dual variables; if no column with negative reduced cost is found, the heuristic
pricing algorithms are used in sequence, until one of them ﬁnds out columns with
negative reduced cost. If all of them fail, the exact pricing algorithm is executed and
all non-dominated columns with negative reduced cost found in this way are inserted
in the RLMP (multiple pricing).
4.3.1 Greedy algorithm
The greedy constructive heuristic computes tentative feasible solutions by visiting the
vertices in a “nearest neighbor” fashion starting from the depot and coming back to it
when no more vertices can be added to the route in a feasible way. The algorithm takes
into account capacity constraints and time windows. For this last purpose it considers
the soft time windows as if they were hard constraints. Starting from the last reached
vertex i ∈ N the next vertex j ∈ N is chosen as the unvisited vertex which minimizes
c
ij − λj and such that it can be visited not later than bj and without exceeding the
capacity.
The greedy constructive heuristic always yields feasible solutions for the VRPTW
and hence for the VRPSTW. If such a solution is better than the best incumbent solu-
tion, all columns corresponding to its routes are inserted into the RLMP , including
those with non-negative reduced cost. This is very useful in early iterations, when
the RLMP contains only the columns produced by the initialization step. The effect
is to stabilize the values of the dual variables and to reduce the optimal value of the
RLMP by a significant amount. The more populated the RLMP is, the less effective
the greedy heuristic becomes. For this reason this heuristic pricing algorithm is used
only when the RLMP does not contain more than 1,000 columns.
4.3.2 Local search
We developed a “best improve” variable neighborhood descent algorithm, using two
neighborhoods DELETE and INSERT. The former which is smaller is explored ﬁrst;
if no improving solution is found, then the latter is explored.
The DELETE neighborhood is made by all routes that are obtained from a current
route f , by deleting one of the customers in it and rescheduling the service starting
times in an optimal way, without changing the order in which the other customers are
visited.
The INSERT neighborhood is made by all solutions obtained from the current route
f by inserting in the route one of the customers not already in it and rescheduling the
service starting times optimally, keeping the order of the other customers unchanged.

5 Branch-and-price
5.1 Search strategy
The search tree is explored with a best-ﬁrst-search strategy, in which subproblems are
evaluated according to the value of their associated lower bound.
The main advantage is that the combination of best-ﬁrst-search with the primal
heuristic described in Sect. 3.1 allows to produce good feasible solutions early in the
search. In turn this allows the removal from the RLMP of all columns whose reduced
cost is larger than the difference between the primal and the dual bounds.
The main drawback is that it is not possible to solve a RLMP in a node of the search
tree starting from the optimal basis of the predecessor node, because this would require
to store explicitly all basic columns for each node. Furthermore nodes of the search tree
solved consecutively have different ﬁxed variables because they come from different
branching decisions.

6 Experimental results
The algorithm was coded in C, compiled with gcc and run on a Pentium IV PC with
1.6 GHz CPU and 500 MB RAM, Linux operating System. The allotted computing
time for each instance was limited to 1 h and the amount of available memory was
limited to 500 MB, to avoid swap operations.
6.1 Instances
For our experiments we used instances derived from the well-known Solomon’s
VRPTW benchmark ( Solomon 1983 ). These instances are divided into six groups
and they have been designed to put in evidence different factors which can make
VRPTW instances more or less difﬁcult: geographical dispersion of the customers,
average number of customers to be served by each vehicle, fraction of customers with
significantly tight time constraints, width and position of time windows. The custom-
ers’ positions are generated at random in groups R1 and R2, they are clustered in
groups C1 and C2 and they are hybrid in groups RC1 and RC2. In groups R1, C1
and RC1 the vehicles have a capacity allowing to visit a few customers, about 5 to
10 for each vehicle. In groups R2, C2 and RC2 the vehicles have a very large capac-
ity and they may serve more than 30 customers each. In each group the customers
are located in the same way: the instances are different only for the time windows.
All the original Solomon’s instances have 100 customers; as proposed by Kohl et al.
(1999) smaller instances with 25 and 50 customers are obtained by considering only
the ﬁrst customers for each group. Costs and traveling times associated with the arcs
are equal and they are computed by rounding down the Euclidean distance with one

7 Conclusions
In this paper we have presented an exact optimization algorithm for the VRP with
Soft Time Windows. The algorithm is a branch-and-price, based on a reformulation
of the VRPSTW as a set covering problem with additional constraints. The pric-
ing problem, the Resource Constrained Elementary Shortest Path Problem with Soft
Time Windows, is solved via bi-directional and bounded dynamic programming with
decremental state space relaxation. We have described how an inﬁnite number of
non-dominated states are represented in a compact way by means of piecewise linear
functions, whose description is stored in the labels extended by the dynamic program-
ming algorithm; we have also presented the corresponding dominance rules and other
algorithmic details. The branch-and-price algorithm also includes primal heuristics,

A column generation algorithm for the vehicle routing problem 81
The experimental results on the Solomon’s data-set conﬁrm that the problem with
soft time-windows is definitely more difﬁcult than its counterpart with hard time
windows, due to the solution space which is enormously larger. The algorithm is able
to solve only some of the instances with 100 customers, while a similar approach
allows to solve almost all of them in the case of hard time windows ( Salani 2006 ).
The relative difﬁculty of the VRPSTW instances has been evaluated by comparing the
results obtained with different values of the penalties for early or late visits.
The algorithmic techniques illustrated in this paper are meant to provide a basis for
the development of decision support systems able to take into account the large variety
of features and constraints arising in real routing problems in distribution logistics.
Time windows have been so far considered as hard constraints in the related scientiﬁc
literature, with a very few exceptions, all related to heuristics. The contribution offered
by an exact mathematical optimization method for the VRPSTW is a step forward to
bring mathematical programming models and algorithms more and more adherent to
the needs of logistic managers and transportation companies.
